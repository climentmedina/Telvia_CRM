# Telvia CRM — Project Brief

## What To Build

A fully interactive CRM web application using **Next.js** (App Router) + **Supabase** + **Tailwind CSS** for exploring, filtering, and acting on ~57,000 scored Spanish company websites. This is a sales prospecting tool for a website optimization/redesign agency. Data is generated by an automated pipeline running on n8n — new records flow in continuously.

Think **HubSpot-level UX** but laser-focused on one use case: identifying Spanish companies with weak websites and reaching out to them.

---

## The Data Pipeline (Context)

An automated 3-tier pipeline analyzes ~67,000 Spanish company websites:

```
CSV (70k companies)
    ↓
[Tier 1] Visit each website, extract 14 heuristic signals     FREE
    ↓ 56,994 reachable (84%)
[Tier 2] Claude Haiku AI analyzes HTML → structured JSON       ~$90 total
    ↓ ~23k done, ~34k processing (finishes ~March 9)
[Tier 3] Claude Sonnet deep analysis + Spanish outreach email  ~$0.20/site, selective
    ↓
[Scoring] Priority score 0-100, tier assignment                FREE
```

The pipeline runs 24/7 on an n8n instance. New Tier 2 results arrive every 5 minutes (50/batch), and scoring follows every 30 minutes. The CRM should handle this live data gracefully.

## Scoring Formula

```
priority_score =
  (improvement_potential / 5) × 30        -- 30 pts max: how much value we can add
+ ((6 - cta_quality) / 5) × 15           -- 15 pts max: poor CTA = better prospect
+ ((6 - value_prop_clarity) / 5) × 12    -- 12 pts max: unclear value prop = room to improve
+ has_contact × 15                        -- 15 pts: has contact form or phone (binary)
+ size_bonus                              -- 12 pts max: micro=12, small=10.8, medium=7.2, large=2.4
+ ((10 - trust_signals_count) / 10) × 8  -- 8 pts max: fewer trust signals = obvious fix
+ is_reachable × 5                        -- 5 pts: site is live
+ has_ssl × 3                             -- 3 pts: basic maintenance proxy
+ (detected_language == 'es') × 5         -- 5 pts bonus: Spanish site
+ (cms in wix/godaddy/jimdo) × 3         -- 3 pts bonus: DIY CMS = more likely to need help
```

**Tier thresholds:** hot ≥ 90, warm ≥ 75, cold ≥ 60, disqualified < 60

**Current distribution (~23k scored):** ~41% hot, ~50% warm, ~8% cold, ~1% disqualified

## Data Quality Notes

- **~85% of sites are reachable** (56,994 of 66,890). The rest failed Tier 1 (timeout, DNS, etc.)
- **~86% detected as Spanish** language
- **~71% run WordPress**, followed by custom, Wix, Shopify
- **Tier 2 failure rate: ~2.7%** (API errors, unparseable HTML)
- **Sectors are in Spanish** (free text from Haiku): "distribución agrícola", "manufactura", "logística", etc.
- **CNAE codes** are Spanish industry classification (like SIC codes) — useful for filtering
- **Provincia** values are Spanish province names: Madrid, Barcelona, Valencia, Sevilla, etc.
- Records with `tier2_status != 'completed'` won't have Tier 2 signals or scores — handle nulls gracefully
- Some records have `sector = 'unknown'` from a past bug (fixed now, but ~100 legacy records remain)

## Core Features

### 1. Dashboard (Home)
- Pipeline progress bars: Tier 1/2/3 completion with live counts
- Score distribution histogram
- Key metrics cards: total, hot, warm, cold, disqualified
- Recent activity feed (latest scored companies)
- **Auto-refresh every 30 seconds** (data arrives continuously)

### 2. Companies Table (main workhorse)
- Sortable, filterable data table with all key columns
- **Filters** (combinable):
  - Outreach tier (hot/warm/cold/disqualified) with colored badges
  - Provincia (dropdown, all Spanish provinces)
  - Sector (searchable dropdown)
  - Business type (B2B/B2C/B2G/mixed)
  - Company size (micro/small/medium/large)
  - CMS platform
  - Score range (slider)
  - Boolean toggles: has_contact_form, has_phone_number, has_ecommerce, etc.
- **Quick actions** per row: open website, copy phone, mark as "contacted"
- **Bulk actions**: export selection to CSV, bulk status change
- Server-side pagination (57k rows — never load all client-side)
- Save filter presets (e.g., "Hot B2B in Madrid with phone")

### 3. Company Detail Page
- All data in organized sections (identity, CRM data, Tier 1 signals, Tier 2 signals, scores)
- Visual score breakdown showing each factor's contribution
- Link to open their website in new tab
- If Tier 3 exists: roast, recommendations, outreach email
- Notable weaknesses displayed as tags/chips
- Activity/notes timeline (for tracking outreach — can be local state or new Supabase table)
- "Similar companies" sidebar (same sector + provincia)

### 4. Analytics
- Score distribution across all scored companies
- Top sectors by count and average score
- Geographic breakdown by provincia (table or map)
- CMS platform distribution pie/bar chart
- Most common notable_weaknesses (frequency chart)
- Pipeline funnel: total → reachable → scored → hot → contacted

### 5. Outreach Management (simple CRM layer)
- Contact status tracking: new, contacted, replied, meeting, won, lost
- Outreach notes with timestamps
- Kanban board view for pipeline management
- Quick filters: "needs follow-up", "contacted this week", "hot + not contacted"

## Important Constraints

1. **No authentication** — single-user internal tool for now
2. **html_raw is huge** — exclude from all queries except detail page (and even there, only if needed)
3. **Data is LIVE** — handle auto-refresh, optimistic updates, stale data gracefully
4. **Supabase service_role key** goes in `.env.local` as `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` (server-side only for the service key)
5. **CNAE codes** are hierarchical (e.g., "47" = retail, "47.1" = retail in non-specialized stores) — consider grouping
6. **notable_weaknesses** JSONB contains arrays of varying length — design UI to handle 0 to 10+ items
7. **Contact status / outreach notes** don't exist in the schema yet — you'll need to either add columns to `companies` or create a new `outreach_notes` table. Prefer a new table to keep the pipeline schema clean.
